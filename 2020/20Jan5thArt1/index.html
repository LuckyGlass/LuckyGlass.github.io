OI题解 - Count Subsequences(Codechef)

什么时候信息竞赛也要期末考试了……

# 题面

> Codechef传送门

如果一个整数序列的各元素之和可以被给定整数 KKK 整除，则称这个序列是好的。给定整数序列 A1,A2,...,ANA_1,A_2,...,A_NA​1​​,A​2​​,...,A​N​​。Hasan想计算该序列有多少子序列是好的。不过这样的话问题就太简单了，因此Hasan决定再加一个限制。

任意非空下标序列 i1<i2<⋯<iLi_1< i_2< \cdots < i_Li​1​​<i​2​​<⋯<i​L​​对应子序列Ai1,Ai2,...,AiLA_{i_1}, A_{i_2},..., A_{i_L}A​i​1​​​​,A​i​2​​​​,...,A​i​L​​​​。如果满足 iL−i1≥Wi_L-i_1\ge Wi​L​​−i​1​​≥W，则称这个子序列非常好。请帮Hasan统计非常好的子序列方案数。由于答案可能非常大，请输出方案数对109+710^9+710​9​​+7 取模的结果。

（多组数据，每个数据点包含 TTT 组数据）

数据规模：
- 1≤T≤10001\le T\le10001≤T≤1000；
- 1≤N≤1051\le N\le 10^51≤N≤10​5​​，0≤W≤N−10\le W\le N-10≤W≤N−1；
- 0≤Ai≤K−10\le A_i\le K-10≤A​i​​≤K−1，2≤K≤502\le K\le502≤K≤50；
- 每个数据点中的所有 NNN 之和不超过 3×1053\times 10^53×10​5​​；

# 解析

如果没有 WWW 的限制，这道题是一道非常简单的 DP，类似于背包。大概也不需要说了。

考虑在 WWW 的限制下怎样求答案。首先我们固定所选子序列的左端点（最左边元素）位置为 LLL，那么就必须要在 [L+W,N][L+W,N][L+W,N] 这段区间内选择至少一个元素。最后把所有位置作为 L=1 NL=1\text~NL=1 N 对应答案求和就可以了，不难发现因为每一次固定的左端点不同，所以这样求和得到的方案数不重不漏。

假设现在确定了 LLL，怎样计算方案数？
设计 DP 状态 f[i][j] 表示已经在 AL+1 AiA_{L+1}\text~A_iA​L+1​​ A​i​​ 这些元素中选取了一些元素，且元素之和模 KKK 余 jjj 的方案数。那么下面的转移应该很好理解：

f_{L,0}=1\\f_{L,1\text~K-1}=0\\ f_{i,j}=f_{i-1,j}+f_{i-1,(j-A_i+K)\bmod K}

啊哈？
然后方案数就是“在 AL+1 ANA_{L+1}\text~A_NA​L+1​​ A​N​​ 中选取的元素之和被 KKK 整除的方案数”-“在 AL+1 AL+W−1A_{L+1}\text~A_{L+W-1}A​L+1​​ A​L+W−1​​ 中选取的元素之和被 KKK 整除的方案数”（这样减了就可以保证一定在 AL+W ANA_{L+W}\text~A_NA​L+W​​ A​N​​ 中选取了至少一个元素了）。

但是不可能枚举 LLL 然后每个都分别计算。

于是题解中所说的分治是这样的：定义“询问区间”为 [L,L+W][L,L+W][L,L+W]。从初始区间 [1,N][1,N][1,N] 开始，求解每个被包含在当前区间中的询问区间 的答案；设当前区间为 [left,right][left,right][left,right] 定义 midmidmid 为区间中点，递归求解 [left,mid][left,mid][left,mid]、[mid+1,right][mid+1,right][mid+1,right]。

求解每个被包含在当前区间中的询问区间 的答案的过程如下：

定义 DP 状态 fl[i][j] 表示在 Ai AmidA_i\text~A_{mid}A​i​​ A​mid​​ 中选出一些元素，这些元素之和模 KKK 余 jjj 的方案数；fr[i][j] 表示在 Amid+1 AiA_{mid+1}\text~A_iA​mid+1​​ A​i​​ 中选出一些元素，……（同上）的方案数。转移就没必要说了。先把这些 DP 值处理出来。

（感觉不是很方便叙述，不如我举个栗子）

  1. 询问区间跨过区间中点：

    那么询问区间的答案就是“必须选L，L+1~mid随便选的方案数”乘“mid+1~L+W-1随便选，L+W~right至少选一个的方案数”。用 DP 值表示就是 (fl[L][j]-fl[L+1][j])*(fr[right][j_]-fr[L+W-1][j_]) （其中 j 和 _j 之和为 K 或 0）。

  2. 询问区间完全在左区间：

    这样无法直接算出完整的答案，但是可以把答案拆成两步计算：
    ①必须选L，在 [mid+1,right][mid+1,right][mid+1,right] 中至少选一个，[L+1,mid][L+1,mid][L+1,mid] 随便选的方案数；
    ②必须选L，不在 [mid+1,right][mid+1,right][mid+1,right] 中选，但是在 [L+W,mid][L+W,mid][L+W,mid] 中至少选一个，[L+1,L+W−1][L+1,L+W-1][L+1,L+W−1] 随便选的方案数；

    可以发现这两个方案加起来恰好能不重不漏地得到所有选择的方案。

    对于 ①，可以用当前的 DP 值计算出：(fl[L][j]-fl[L+1][j])*(fr[right][j_]-fr[mid][j_])；

    对于 ②，可以递归求解：在左区间 [L,mid][L,mid][L,mid] 中求解，一定是能够计算完的。

算法思想就是这样。

复杂度分析其实很简单：因为这样取中点递归，最多会递归 logN\log NlogN 层；每一层计算 fl[i][j] 和 fr[i][j] 都需要枚举 i=1~N，j=0~K-1，均摊 O(NK)O(NK)O(NK)；枚举询问区间在每一层中也是 O(N)O(N)O(N) 的。

因此复杂度是 O(NKlogN)O(NK\log N)O(NKlogN)。
