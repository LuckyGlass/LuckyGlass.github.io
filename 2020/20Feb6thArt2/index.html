OI题解 - Construction of a tree(AtCoder)

发表于 2020-02-06 分类于 OI题解 Valine：

这道题好有意思啊（感觉构造题的方法都特别神奇）

# 题意

有 nnn 个点（1到n）和 n−1n-1n−1 个点集（E1E_1E​1​​到En−1E_{n-1}E​n−1​​），要从每个点集中选出两个不同的点，并将这两个点连边。

是否存在一种选取点的方案使得 nnn 个点与所连的边构成一棵树，如果存在，输出任意一种方案。

数据规模：2≤n≤1052\le n\le10^52≤n≤10​5​​；∣Ei∣>1|E_i|>1∣E​i​​∣>1；∑∣Ei∣≤2×105\sum |E_i|\le2\times10^5∑∣E​i​​∣≤2×10​5​​

# 解析

考虑这样建图：图上的 nnn 个点作为二分图的甲部，n−1n-1n−1 个集合每个集合建一个点，作为乙部；如果点 iii 在集合 EjE_jE​j​​ 中，那么它们之间在二分图上存在一条边。

就有一个结论：“将甲部任意一个点删除后，二分图有完全匹配”是“可以构成树”的充要条件。

> 证明

  1. 必要条件（如果“可以构成树”，则满足该条件）：

    不难发现其实 n−1n-1n−1 个集合就是 n−1n-1n−1 条边。
    假设删除的甲部的点为 uuu；如果“可以构成树”，那么把 uuu 作为这棵树的根，整棵树就会变成下面这样：

    于是每个点与它上面的边匹配，就可以形成完全匹配。

  2. 充分条件（如果满足该条件，则“可以构成树”）：
    考虑在该条件下如何构造一棵树。

    • 先取点 111 作为树根，然后把 111 从二分图中删掉，求一个完全匹配；找到任意一个包含 111 的集合 EiE_iE​i​​，设完全匹配中与 EiE_iE​i​​ 匹配的点为 uuu，就连边 (1,u)(1,u)(1,u)。树的点集 S={1,u}S=\{1,u\}S={1,u} 。
    • 在二分图中删除 u,Eiu,E_iu,E​i​​。显然二分图仍然存在完美匹配。
    • 求完全匹配，找到一个包含 SSS 中任意一个点 xxx 的集合 EjE_jE​j​​，设完全匹配中与 EjE_jE​j​​ 匹配的是点 vvv；连边 (x,v)(x,v)(x,v)。树的点集 S={1,u,v}S=\{1,u,v\}S={1,u,v}。
    • 在二分图中删除 v,Ejv,E_jv,E​j​​。
    • ……（直到删完所有点）

虽然时间复杂度并不允许我们枚举删除一个点，判断是否有完美匹配；但是利用这个结论，可以先删除点 111，然后求完美匹配——如果 uuu 与 EiE_iE​i​​ 匹配就把 uuu 与 EiE_iE​i​​ 中的所有点（除了本身）连边，并且给这些边打上 EiE_iE​i​​ 的标记（这样才能判断连的边是哪一个点集形成的）。

然后从 111 开始 DFS，每种标记的边只能走一条（每个集合只提供一条边）、每个点只能经过一次（树）。如果能够遍历所有点，那么 DFS树 就是一个解；否则无解。
