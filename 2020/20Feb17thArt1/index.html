# 类欧几里得算法

欧几里得算法是计算 (a,b)(a,b)(a,b)，主要思想是 (a,b)=(b\bmod a,a)。这样保证了它的时间复杂度。

而类欧几里得算法，并不是像扩展欧几里得算法那样，它的正确性和欧几里得算法没有任何关系，只是它也有类似于 (a,b)\to(b\bmod a,a) 这种操作，从而保证时间复杂度。

常见的 333 种（然而我并不知道还有没有别的）可以用类欧几里得算法计算的式子有：

\begin{align} &f(a,b,c,n)=\sum_{i=0}^n\left\lfloor\frac{ai+b}{c}\right\rfloor\\ &g(a,b,c,n)=\sum_{i=0}^ni\left\lfloor\frac{ai+b}{c}\right\rfloor\\ &h(a,b,c,n)=\sum_{i=0}^n\left\lfloor\frac{ai+b}{c}\right\rfloor^2 \end{align}

# 算法主体

- 计算 f

当然从看起来最简单的入手~

看到下取整，我们可以先把我们能够提取出的整数部分提取出来——当 a≥ca\ge ca≥c 或 b≥cb\ge cb≥c 时，令 a'=a\bmod c，b'=b\bmod c：

\begin{align} f(a,b,c,n)&=\sum_{i=0}^n\left\lfloor\frac{ai+b}{c}\right\rfloor\\ &=\sum_{i=0}^n\left(\left\lfloor\frac{a'i+b'}{c}\right\rfloor+\left\lfloor\frac ac\right\rfloor i+\left\lfloor\frac bc\right\rfloor\right)\\ &=\sum_{i=0}^n\left\lfloor\frac{a'i+b'}{c}\right\rfloor+\sum_{i=0}^n\left\lfloor\frac ac\right\rfloor i+(n+1)\left\lfloor\frac bc\right\rfloor\\ &=f(a',b',c,n)+\frac{n(n+1)}2\left\lfloor\frac ac\right\rfloor+(n+c)\left\lfloor\frac bc\right\rfloor \end{align}

经过这一步后，有 a<ca<ca<c 且 b<cb<cb<c。在此基础上继续：令 m=⌊an+bc⌋m=\left\lfloor\frac{an+b}{c}\right\rfloorm=⌊​c​​an+b​​⌋

Warning!!! 公式恐惧者慎入

\begin{align} f(a,b,c,n)&=\sum_{i=0}^n\left\lfloor\frac{ai+b}{c}\right\rfloor\\ &=\sum_{i=0}^n\sum_{j=1}^m\left[j\le\left\lfloor\frac{ai+b}{c}\right\rfloor\right]\\ &=\sum_{i=0}^n\sum_{j=0}^{m-1}\left[j+1\le\left\lfloor\frac{ai+b}{c}\right\rfloor\right]\\ &=\sum_{i=0}^n\sum_{j=0}^{m-1}\left[(j+1)c\le ai+b\right]&\text{因为}\small{c\left\lfloor\frac{ai+b}{c}\right\rfloor\le ai+b}\\ &=\sum_{i=0}^n\sum_{j=0}^{m-1}[cj+c-1\left\lfloor\frac{cj+c-b-1}a\right\rfloor\right]\\ &=\sum_{j=0}^{m-1}\left(n-\left\lfloor\frac{cj+c-b-1}a\right\rfloor\right)\\ &=nm-\sum_{j=0}^{m-1}\left\lfloor\frac{cj+c-b-1}a\right\rfloor\\ &=nm-f(c,c-b-1,a,m-1) \end{align}

注意到上式的 a,ba,ba,b 实际上是对 ccc 取模得到的。因此，我们相当于完成了 (a,c)\to(a\bmod c,c)\to(c,a\bmod c)。根据欧几里得算法的证明，可以得到该算法在复杂度上的正确性。
