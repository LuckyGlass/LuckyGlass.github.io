学习笔记 - KMP字符串匹配算法

发表于 2019-11-08 分类于 学习笔记 Valine：

其实之前学过好多次了，都没有学懂……
现在学懂了（写一篇博客免得忘掉）

Part1. 字符串匹配问题

在这个问题中，给出两个字符串 S,TS,TS,T，要求回答 TTT 是否是 SSS 的一个子串。

关于 S,TS,TS,T，在字符串匹配问题中，SSS 称为主串，TTT 称为模式串。那么我们就是要在主串中找模式串。

Part2. Brute-Force

顾名思义，暴力算法。设主串 SSS 长度为 nnn，模式串 TTT 长度为 mmm。那么我们暴力地枚举每个 SSS 的子串（其实就是枚举每个子串的开始位置），复杂度 O(n)O(n)O(n)；然后对于每个子串，暴力检测子串是否与 TTT 相同，复杂度 O(m)O(m)O(m)。

所以复杂度为 O(nm)O(nm)O(nm)。显然太慢了，尤其是主串 nnn 比较大的时候。

Part3. KMP 算法

“KMP” 是算法提出者（不止一个人，但是我忘了 awa）的名字缩写。

Part3/1. 算法思路

首先分析 Brute-Force 的缺陷：完全没有利用上一次匹配得到的信息，每一次匹配要将模式串的指针和主串的指针都回溯到起始位置。

相反，KMP 就尽可能地利用了已有的匹配信息，并且***不再回溯主串的指针***，而是把 模式串向右移动。而且模式串向右移动并不是简单的右移一位，而是移动到尽可能远的、可能匹配的位置。

下面这种情况，Si−j+1→i−1S_{i-j+1\to i-1}S​i−j+1→i−1​​ 和 T0→j−1T_{0\to j-1}T​0→j−1​​ 已经匹配，但是在 Si,TjS_i,T_jS​i​​,T​j​​ 这个位置失配，就像下图这样：

那么我们考虑把 T 向右移动，假设移动到 SiS_iS​i​​ 对应 TkT_kT​k​​ 时，Si−k→i−1S_{i-k\to i-1}S​i−k→i−1​​ 和 T0→kT_{0\to k}T​0→k​​ 匹配了。

（并不知道 SiS_iS​i​​ 和 TkT_kT​k​​ 是否会匹配）说明 kkk 是下一个可能匹配的位置。于是我们把 TTT 的指针移动到 kkk 继续匹配。

在 KMP 中，当 k 和 j 满足上述关系时，定义 fail 指针 fail(j)=kfail(j)=kfail(j)=k。

Part3/2. fail 指针的性质

考虑 kkk 和 jjj 满足什么条件，fail(j)=kfail(j)=kfail(j)=k。

框起来的这部分一定完全一样，不然 T0→k−1T_{0\to k-1}T​0→k−1​​ 不可能和 Si−k→i−1S_{i-k\to i-1}S​i−k→i−1​​ 匹配。也就是 T0→k−1=Tj−k→j−1T_{0\to k-1}=T_{j-k\to j-1}T​0→k−1​​=T​j−k→j−1​​，所以 fail 指针其实和主串 SSS 没有关系，只跟 TTT 本身相关。

根据定义，以及算法的特殊设计：

fail(j)=\begin{cases} -1&(j=0,算法的特殊设计)\\ \max\{k|T_{0\to k-1}=T_{j-k\to j-1}\}\\ 0&(如果 i\not=0且不存在满足条件的k) \end{cases}

如果 i=0i=0i=0，即模式串的开头就失配了，就只有把模式串向右移一位了，于是 fail(0)=−1fail(0)=-1fail(0)=−1。如果存在这样的 kkk，就可以将模式串向右移动到尽可能远的位置（所以是取 max\maxmax）。如果不存在这样的 kkk，就只有把模式串挪到开头，尝试匹配。

Part3/3. KMP 主函数

有了 fail 指针，我们就可以不再回溯主串的指针，而是将模式串向右挪到 fail()fail()fail() 的位置。

Part3/4. 求 fail 指针

最难计算的显然是 fail(j)=max{k∣T0→k−1=Tj−k→j−1}fail(j)=\max\{k|T_{0\to k-1}=T_{j-k\to j-1}\}fail(j)=max{k∣T​0→k−1​​=T​j−k→j−1​​} 的情况嘛。

假如我们已经计算出来了 fail(j)fail(j)fail(j)，怎样递推的求出 fail(j+1)fail(j+1)fail(j+1)？

  1. 如果 Tfail(j)=TjT_{fail(j)}=T_{j}T​fail(j)​​=T​j​​，那么就可以向后匹配一位，也就是 fail(j+1)=fail(j)+1fail(j+1)=fail(j)+1fail(j+1)=fail(j)+1；
  2. 否则就相当于把 TTT 同时作为模式串、主串进行匹配，在 Tfail(j)T_{fail(j)}T​fail(j)​​ 和 TjT_jT​j​​ 位置失配，照样采用 KMP 的思想，将 j 跳回 fail(j)fail(j)fail(j) 的位置，继续匹配，直到匹配成功或者退回到起始位置；
